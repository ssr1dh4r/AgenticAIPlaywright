# Test Plans

Comprehensive test plans for each QE AI Kanban ticket.

## Files

### KAN-1-test-plan.md
**SauceDemo CheckOut Validation**
- 34 test cases across 5 acceptance criteria
- Covers: Cart review, checkout info, order overview, completion, error handling

### KAN-2-test-plan.md
**SauceDemo Login Validation**
- 57 test cases organized by category
- Covers: Successful login, invalid credentials, UI validation, edge cases, security

### KAN-3-test-plan.md
**EA API Endpoint Validation**
- 72+ test cases for REST API endpoints
- Covers: Authentication, CRUD operations, validation, error handling, security

## Format

Each test plan includes:
- Test ID (KAN-X-ACY-TCZ)
- Test title (self-explanatory)
- Description
- Preconditions
- Steps
- Expected result
- Test data

## Usage

These test plans serve as:
1. **Requirements** - Acceptance criteria definition
2. **Documentation** - For stakeholders and team
3. **Automation basis** - Source for automation scripts
4. **Coverage reference** - What's being tested
